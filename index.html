Oh My Stories!


Description: This app is a multi-step form split into several pages, through which the user navigates in order to generate a story. I designed it based on TurboTax, which has a lot of branching steps and remembers user input if users want to go back and revise their answers to previous questions. I initially used JavaScript to hide and show the question fields one at a time, but realized it would be much more engaging if the story-generator form was dynamic and each subsequent question displayed relevant user input from previous questions. For example, I wanted questions 2-15 to reference the name of the character that the user provided in question 1, and so on.

After trying out many failed approaches, talking to friends at Women Who Code DC meetups, chatting with others in the CodeNewbies slack channel about my goals, and finally watching a relevant RailsCast episode, I arrived at the current approach, in which each question is a partial. The story.rb model knows about the questions and which question is current, so I’m using that information to dynamically change which partial is rendered in the “new” view. The questions method in the story model returns an array of questions listed in the correct order, and the next_question method sets the next question equal to the the index number of the current step, plus one.

The app passes from one question to the next (building a story) through the new and create controller actions. The form remembers state and user input inside a session.

Ultimately I want to store all the user’s stories in the db and make stories accessible only to the users who created them. For this reason, my next goal is to add Devise and update the controller new and create actions so that stories are built out from the current user.

Technologies: Ruby, Ruby on Rails, Postgres, HTML5, CSS3, CSS animation, JavaScript, Bootstrap, Pixelmator

Link: www.ohmystories.com
